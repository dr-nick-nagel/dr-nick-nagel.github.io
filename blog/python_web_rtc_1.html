Using the WebRTC Session Description Protocol in Python

Introduction

If there's one thing I've learned over the years it's all about the importance of standards ...

Over the course of researching my needs for a WebRTC project I'm presently working on I came across a post with a bit of annoying misinformation which I do want to take the opportunity to clarify. The author asserted that WebRTC is not a protocol but rather that it is a "project" and they went on to claim further that this "project" is owned by Google[1]. I apologize here but I pride myself on writing with accuracy to the greatest extent possible -- especially where technical matters are concerned -- and these kinds of false assertions do make me bristle a bit. So please indulge me while I do my best to clarify.

WebRTC comprises a standard (a set of standards actually) which use a number of protocols and Application Programmer Interfaces (API's) intended to facilitate communication across a wide range of devices and support a wide range of use-cases. The standards adopted by WebRTC are open standards meaning that they are not "owned" by any one individual or company. WebRTC protocols and definitions may be adopted by anyone anwhere who is willing and able to provide a compliant implementation -- and indeed the purpose of this post is to explore and facilitate the use of one particular implementation provided for the python programming language. Which is open source. And free.

Session Description Protocol

SDP is intended to enable sharing arbitrary data between peers. Data can be anything from simple messages exchanged across the Internet of Things to video streams, multiplayer game objects and more. SDP is just one part of WebRTC, but it serves the critcal role of facilitating the exchange of information necessary to enable the exchange of data.

Personally, I am a visual thinker and so I like to create visualizations to better understand the technologies I work with. The diagram below is intended to illustrate the protocol.

Figure i. Session Description Protocol (SDP).

A WebRTC session begins with SDP. Agents in the exchange (which I'll refer to here as sender and receiver) exchange metadata necessary for communication between the participants to take place. The WebRTC abstraction to handle the information exchange is defined by the RtcPeerConnection interface. The interface defines local and remote session description attributes which hold information participating agents need to enable transmission. At a high level the SDP exchange (part of the signalling process) proceeds as follows.

  1. The sender creates an offer which includes information the receiver will need in order to receive the transmission and sets it as its local description.

  2. The receiver, on receiving the offer, sets it as its remote description and generates an answer which it sets as its local description and sends back to the sender.

  3. On receiving the answer the sender sets it as its remote description.

Over the course of the SDP exchange the peer connection objects enter into a number of states as illustrated below.

new --> connecting --> connected ...

The initial state is new. During the offer / answer exchange peer connection objects enter into the connecting state. Once both objects have processed both offer and answer (by setting the local and remote session description object attributes) the PC's enter into the connected state and are ready for transmission.
