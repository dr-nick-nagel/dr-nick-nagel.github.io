First, what exactly are SVG filters?

SVG filters are procedural effects that can be applied to create SVG artwork that:

  1. Is natively textured, and

  2. Exhibits a wide range of effects including blur, lighting effects, color manipulation and even more complex features and distortions you might imagine as an artist.

How SVG Filters Work

You can create SVG filter effects using the <filter> element. Filters are defined by compositing SVG filter primitives -- the "building blocks" of filter effects (see Appendix 1). The primitives define atomic graphics operations (like blur, lighting effects, displacements, etc.). The output of these operations are termed results.

Filters operate on inputs; the original source graphic or the result of prior filters in the pipeline. Source graphics can be images, SVG shapes, groups, etc.. Inputs to primitives targeting graphics elements can be specified using the keyword SourceGraphic.

! --- Definition admonition SourceGraphic: A keyword referring to the graphics elements that were the original input into the 'filter' element.

Apendix 1: List of SVG Feature Primitives

SVG defines a large number of filter primitives which can be used to create an infinitude of visual effects. The standard filters include:

  1. : Composites two input images using a blend mode.

  2. : Applies a color matrix to the input image, allowing for changes to hue, saturation, brightness, and contrast.

  3. : Performs a per-channel color manipulation using a function. It's often used for advanced color adjustments.

  4. : Composites two input images using a compositing operator like over, in, out, or atop.

  5. : Applies a matrix convolution filter, useful for effects like blurring, sharpening, and embossing.

  6. : Lights a graphic by using the alpha channel as a bump map.

  7. : Displaces pixels in an image based on the pixel values of a second image.

  8. : Creates a drop shadow effect. (This is a simplified primitive that combines other primitives.)

  9. : Fills the filter subregion with a solid color and opacity.

  10. : Applies a Gaussian blur to the input image.

  11. : Fetches image data from an external source to be used as an input for other primitives.

  12. : Combines multiple input images into a single output image.

  13. : A child element of that specifies an input for the merge operation.

  14. : Changes the shape of an image's alpha channel, often used for thinning or thickening a shape.

  15. : Shifts the input image by a specified amount.

  16. : Creates a lighting effect, with a glossy, reflective surface.

  17. : Fills a region with a repeating pattern from a source image.

  18. : Creates an image using the Perlin turbulence function, which is useful for generating procedural textures like clouds, marble, or fire.
